🔄 跳題問題修復測試

🎯 問題描述

❌ 觀察到的問題

用戶反映："在選擇完選項後做切換中英文，會出現一次跳兩題的問題，例如我在第四題答題完做切換，第五題還沒出來，他會直接跳到第六題"

  • ✅ 正常流程：第4題答題 → 語言切換 → 進入第5題
  • ❌ 異常流程：第4題答題 → 語言切換 → 跳過第5題 → 直接進入第6題
  • ❌ 這會導致用戶錯過題目，影響測驗完整性

問題原因：在語言切換邏輯中，索引更新和題目顯示的時機不當，導致跳過了一題。

🔧 解決方案

✅ 修復邏輯

修復前（有問題）：
1. 第4題答題完成
2. 語言切換觸發
3. 在 language-switcher.js 中更新索引到第5題
4. 調用 displayCurrentQuestionAfterLanguageSwitch
5. 函數使用更新後的索引（第5題）
6. 但由於某些邏輯問題，實際顯示第6題 ❌
↓
修復後（正確）：
1. 第4題答題完成
2. 語言切換觸發
3. 調用 displayNextQuestionAfterLanguageSwitch
4. 函數內部更新索引到第5題
5. 載入新語言題目文件
6. 更新整個 selectedQuestions 數組
7. 顯示第5題（新語言版本） ✅

⚙️ 技術實現詳情

核心修復點：

1. 創建新的函數 displayNextQuestionAfterLanguageSwitch()
   - 專門處理已作答情況下的語言切換
   - 在函數內部處理索引更新
   - 確保邏輯順序正確

2. 修改 language-switcher.js 中的調用邏輯：
   - 已作答情況：調用 displayNextQuestionAfterLanguageSwitch()
   - 未作答情況：調用 displayCurrentQuestionAfterLanguageSwitch()

3. 確保索引更新的原子性：
   - 索引更新和題目顯示在同一個函數中完成
   - 避免外部更新索引導致的時序問題

關鍵代碼變化：

// 修復前（有問題）
if (isAnswered) {
  window.setCurrentQuestionIndex(currentIndex + 1);
  await window.displayCurrentQuestionAfterLanguageSwitch();
}

// 修復後（正確）
if (isAnswered) {
  await window.displayNextQuestionAfterLanguageSwitch();
}

// 新函數 displayNextQuestionAfterLanguageSwitch()
async function displayNextQuestionAfterLanguageSwitch() {
  currentQuestionIndex++; // 在函數內部更新索引
  // 載入新語言題目文件
  // 更新 selectedQuestions 數組
  displayQuestion(); // 顯示更新後的題目
}

🧪 測試步驟

📋 詳細測試流程

  1. 準備測試環境
    • 前往筆試測驗頁面
    • 開啟開發者工具 Console 標籤
    • 清空 Console 歷史記錄
  2. 測試跳題修復
    • 開始測驗
    • 回答第1、2、3題
    • 在第4題答題完成後，立即點擊語言切換按鈕
    • 觀察是否正確進入第5題（而不是跳過到第6題）
    • 檢查 Console 中的調試信息
  3. 測試多次跳題
    • 繼續回答第5、6題
    • 在第7題答題完成後，再次點擊語言切換按鈕
    • 觀察是否正確進入第8題
    • 重複測試幾次，確認修復穩定
  4. 測試不同題目位置
    • 在不同題目位置測試語言切換
    • 確認修復在所有題目位置都有效
    • 特別測試接近測驗結束的情況
