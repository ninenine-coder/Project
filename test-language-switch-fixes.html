🔧 語言切換修復測試

🎯 問題描述

❌ 觀察到的問題

用戶反映兩個主要問題：

  • 問題1：選項數量異常 - "我在第一題顯示題目時，做切換中英文就會出現8個選項，且四個選項為中文，四個選項為英文"
  • 問題2：按鈕啟用時機錯誤 - "開放中英文切換按鈕是在顯示題目就開放，而不是等選項顯示後才開放，切換後一樣先顯示題目，在顯示選項"

🔧 解決方案

✅ 修復方案

問題1修復：防止8個選項問題
1. 創建新的函數 displayQuestionForLanguageSwitch()
2. 該函數只顯示題目文字，不創建選項
3. 清空選項容器，防止重複選項
4. 在語言切換時使用此函數而非 displayQuestion()
↓
問題2修復：正確的按鈕啟用時機
1. 在 displayQuestionWithDelay() 中，顯示題目後立即啟用語言切換按鈕
2. 在 displayQuestionForLanguageSwitch() 中，顯示題目後立即啟用語言切換按鈕
3. 確保語言切換按鈕在題目顯示時就可用

⚙️ 技術實現詳情

核心修復點：

1. 新增函數 displayQuestionForLanguageSwitch()：
   - 只顯示題目文字
   - 清空選項容器（防止重複選項）
   - 隱藏計時器
   - 立即啟用語言切換按鈕

2. 修改 displayQuestionWithDelay()：
   - 在顯示題目文字後立即啟用語言切換按鈕
   - 確保用戶可以在題目顯示時就進行語言切換

3. 修改 displayCurrentQuestionAfterLanguageSwitch()：
   - 使用 displayQuestionForLanguageSwitch() 而非 displayQuestion()
   - 避免創建重複選項

關鍵代碼變化：

// 新增函數
function displayQuestionForLanguageSwitch() {
  // 顯示題目文字
  document.getElementById('question-text').textContent = 
    `${currentQuestionIndex + 1}. ${question.question}`;
  
  // 清空選項容器（防止重複選項）
  const optionsContainer = document.getElementById('options');
  optionsContainer.innerHTML = '';
  
  // 立即啟用語言切換按鈕
  const languageSwitcher = document.querySelector('.language-switcher');
  if (languageSwitcher) {
    languageSwitcher.style.pointerEvents = 'auto';
    languageSwitcher.style.opacity = '1';
    languageSwitcher.style.cursor = 'pointer';
  }
}

// 修改 displayQuestionWithDelay()
// 在顯示題目後立即啟用語言切換按鈕
const languageSwitcher = document.querySelector('.language-switcher');
if (languageSwitcher) {
  languageSwitcher.style.pointerEvents = 'auto';
  languageSwitcher.style.opacity = '1';
  languageSwitcher.style.cursor = 'pointer';
}
